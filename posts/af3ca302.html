1. 两数之和

https://leetcode.cn/problems/two-sum/

思路

  1. 暴力解法，两层for循环遍历数组，然后在数组中找和为target的元素 同时注意两个值不能为同一个数组元素【不能是同一个数组下标】因为题目中写道同一个元素不能在答案中重复出现

  2. 这里要将题目进行转换，求a+b = target 转换为 根据a 找到数组中有没有存在b=target-a 这个值的存在

    1. 如果存在 需要暂存b对应的索引位置
    2. 这里不要求有序 但是不仅需要对应的数组value 还要活得该值在数组中的下边，所以考虑使用HashMap存储
    3. 使用map的话 要根据数组的value值寻找元素 所以要将数组的value作为map的key 同时将数组的index 作为map的value 存储
    4. 当没有找到对应的b时候，将数组的索引和value同时放入map中，并循环找下一个数组元素

  3. 如下图解析

本题其实有四个重点：

  • 为什么会想到用哈希表
  • 哈希表为什么用map
  • 本题map是用来存什么的
  • map中的key和value用来存什么的

把这四点想清楚了，本题才算是理解透彻了。

自己实现

收获：

  1. hashMap 的操作
  2. 怎样使用哈希表+数组 对逻辑进行实现以及怎样去重

参考

第15题. 三数之和

https://leetcode.cn/problems/3sum/

题目中要求和为0 且三个num[i] 不能重复

  1. 可以使用a+b+c = 0 来进行判定 先确定a 在确定b+c= 0-a 使用三层for循环+哈希表来进行确定值【类比于两数之和】
  2. 但是要注意要求 是所有的三元组不能重复，需要判定不同的值 来将元素去重

思路

  1. 使用三层for循环 依次 来循环遍历数组元素 找到对应a b c 来求得不同的三元组
    1. 对数组先进行排序，因为默认要求target=0 所以必须有正数和负数都存在（最大值小于0 或者最小值 大于0 都不符合题意）
    2. 可以使用哈希表来存储对应的下标和元素值
    3. 但是要求去重 可能会有很多边界条件限制
  2. 使用双指针法，对排序数组 使用两次循环 ，降低时间复杂度
  3. 去重的逻辑【a b c 对应不同的操作】
  4. ==这里要求返回的是对应的元素值 而不是索引 所以可以对数组进行排序而不用保留索引值==

自己实现

一上午的新路历城: 从早上9点开始接触三数之和 一直到下午两点 中间吃完午饭+午休后 开始总结一下

一开始代码随想录中 没有java解法，只有c++代码 ，就想着去公布的题解中找一个对应的java代码

  1. 终于找到了: 15. 三数之和，Java简洁题解（暴力、hash、双指针） - 三数之和 - 力扣（LeetCode）
    1. 题解中有个hash的加速没成功 但是下一个评论中成功了 就开始想为什么会失败
    2. 最终找到了原因: 是因为没有把0,0,0， 这种情况考虑进来 把去重放到了外边导致一直死循环 需要在找到一个结果后再去重，而不是直接对第二个数字去重
  2. 接着看到了其他的题解: 暴力解法 有一个题解是用的二分法查找 来减少第三个数据的时间 能提交通过
  3. 还有一个题解: 使用map来进行去重的同时 在map中存放 0-a-b 然后在遍历存放b的时候直接查找c 而不是存放c 在查找的时候找0-a-b
  4. 看到labuladong方法 总结了一下怎样解决n数之和 ，还需要自己动手写代码去提高双指针的思路一个方法团灭 nSum 问题 :: labuladong的算法小抄

下边题解上一个关于hashmap去重的评论的解释

15. 三数之和，Java简洁题解（暴力、hash、双指针） - 三数之和 - 力扣（LeetCode）

去重的逻辑

这里去重的时候 【-1，-1,2】 这种情况 当num[i] 指向第一个-1 的时候，应该判定后边的情况看是否符合a+b+c=0

当第一个-1判定结束之后，num[i] 指向第二个-1 应该判断前边的值-1 是不是已经检验过一次 此时前边也是-1 表明这种情况出现过 所以直接跳过

这里num[i] 是针对a 的情况 先确定a 在确定 b+c=0-a
