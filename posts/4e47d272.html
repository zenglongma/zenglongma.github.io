一、《峰迷商城》项目介绍

1.1 项目背景

1.1.1B2C
1.1.2C2B2C
1.1.3Java

1.2项目功能

http://www.processon.com/view/link/606bde8b1e08534321fd2103

1.3 技术选型

SSM企业开发框架 比较基础的开发技术

1.3.1 单体项目

项目的页面和代码（交互和数据存取） 都在同一个项目中，项目开发完成之后都会部署在同一个服务器中

单体项目的问题： 用户对页面的静态资源以及java代码（用户登录需要使用控制器 控制器使用java写的，所以需要用到tomcat上的jvm运行环境进行运行，所以需要请求tomcat）的请求压力都会落在Tomcat服务器上

单体项目 不能实现高并发等

1.3.2技术清单

  • 代码块 Ctrl + Shift + K（或者 ``` (三个esc下边的按键（英文输入法）)

  • 快捷键 ctrl + shift + ] ，生成列表(使用星号 *、或加号+ 或 减号 - 作为列表标记，后添加一个空格，生成列表‍)(-[空格]空格 文字)

  • 项目架构：前后端分离（前端和后端会在两个服务器中，然后实现服务器之间通讯交互）

  • 前端技术:（JQuery不再使用，有渲染和交互功能，但是过于笨重）使用轻巧的技术

    • ==vue （渲染） 、axios(通讯)==、妹子uI 、 layui、bootstrap
    • 后端技术：SpringBoot（整合SSM）+MyBatis+ RESTful(开发风格 技术规范)+Swagger（接口呈现）

  • 服务器搭建部署：Linux Nginx（前端）

二、项目的架构演进

互联网项目的三高

高并发高可用高性能

2.1单体架构

  • 特点：前后端都部署在同一台服务器上（前后端代码都在同一个应用中）
  • 缺点：对静态资源的访问压力也会落在Tomcat上

2.2前后端分离

  • 并发性比较高
  • 前后端分离：前端和后端分离进行开发和部署（前后端部署在不同的服务器上）
  • tomcat服务器只负责数据校验，所有的页面跳转等 都是使用Nginx进行（两台服务器同时提供服务，对于页面的请求不会占用tomcat服务器）
  • 优点：（只需要进行接口调用）将对静态资源的访问和对接口的访问进行分离，Tomcat服务器只负责数据服务的访问（可以实现更多人的访问 即使300的并发tomcat 但也能让500人同时访问）

2.3 集群搭建（分布式）

集群在理论上可以解决并发问题 ，还可以解决可用性问题（防止有的服务器出现问题，其他节点可以帮忙）

负载均衡： 按需分配

轮询: 一次一人一个，轮流交替

权重分配:你占比大，你一次2个 我占比小，我一次一个

集群可以解决 高并发问题，还解决了可用性问题

可以有替代服务器进行可用性保护

2.4分布式（高可用性）

2.4.1分布式锁

redis实现分布锁，临时存储要操作的数据，其他服务器若并发操作，则等待锁释放后在进行操作

  • 出现商品超卖问题
    • 使用redis 谁操作库存中某一个数据 谁把改数据的id 写入到redis中 （相当于加锁操作）
    • 三个服务器同时查询一个数据库中的商品，但因为是并发，没有写回数据库导致库存不足
  • 一个服务器的话，不会出现问题，使用sql-net锁
  • 使用分布式技术控制服务器之间的并发问题
2.4.2分布式数据库

使用mycat数据库负载均衡

数据库默认连接最大是151个

多个数据库并发存储（分布式数据库）

2.4.3 ridis集群

redis 也是用集群 保证高可用性，使用负载均衡 cUster 来保证

2.4.4ElasticSearch数据库临时查询
（like模糊查询效率低）

模糊查询 在直接访问数据库的时候会比较慢（数据量大）

  • 使用ElastSearch 来专门实现数据库检索 增加效率

  • 数据库中间件

  • 消息中间件（消息队列）

2.5微服务架构（高性能）

微服务器主要是提高可用性的目标（聊天系统崩了不会影响看视频模块）

可以针对具体的模块去解决高并发问题，从更小的层面提高性能（保证了各个模块的可用性）

将原本一个应用中开发的多个模块进行拆分，单开进行开发和部署（鸡蛋不放在一个篮子里边）

保证可用性和性能

一个模块故障将导致整个节点不可用

为了保证系统的扩展和功能堆叠，会不断增加新的功能（但是整体的坏的可能性变高 一个坏了都坏了）

现在为了避免一个影响整体，将模块进行拆解，各个模块拆解成各个项目，一个模块实现成一个项目

每个模块都有自己的服务器 并且不同请求 对应不同的服务器

对于某个模块需要高并发的话，可以转向某一个模块实现集群部署，来提高该模块的性能，从而减少资源浪费

微服务项目对技术实现变得困难和复杂，对模块之间的数据通信等要求变高，更加专注于系统==可用性==问题，不在过度关注==并发问题==，也提高了系统的==高性能需求==

三、峰迷商城项目进行搭建

3.1技术储备

分为两个部分组成 前端 和后端

==基于maven的聚合工程完成项目搭建，前端采用vue+axios 后端采用SpringBoot整合SSM==

3.2项目搭建

先去完成SpringBoot 整合ssm 学习完成Maven聚合工程，再去学习项目搭建

3.2.1在服务器主机上运行起来一个项目需要的几个步骤

SSM 手动整合工程存在的问题——配置复杂且繁琐

  1. 安装JDK
  2. 安装Tomcat
  3. 安装mysql
  4. 打包ssm项目
  5. 部署项目: 将war包放到Tomcat中
  6. 启动tomcat项目

在之前的ssm项目中 都需要完成固定步骤且繁琐的一些配置工作，没有含金量但是却比较固定且浪费时间

所以需要SpringBoot进行简化开发和部署，方便把精力都放到项目开发上去

springboot这种技术 就是在要满足这种需要的时候演进出来的，所以需要不断发展

SpringBoot就是在两个方面进行简化开发过程

  1. 简化配置步骤，甚至可以直接实现0配置就可以来进行项目开发
  2. 项目部署简单，可以实现项目快速部署，甚至在不需要安装tomcat的时候就可以直接部署，因为SpringBoot内置了tomcat

3.3 Springboot简介

springboot 面临着动态语言 node.js 等挑战 这些语言都是在java的思想基础上做出的改进，所以比较先进 ，开发出了java的春天springboot 框架，内置了很多需要配置项，并且都是主流的配置文件，不需要开发者手动去整合配置，需要的只是在用到的时候直接引用一下即可，这样用习惯大于配置的思想 为开发者简化开发流程

==上边是简略图==

注意点:

  1. springboot 不是没有配置，只是内置了配置
  2. springboot 是云原生的基础，为后边springcloud 做了准备
  3. 面对缺点的时候，可以用开发者的能力去弥补springboot的高内置，用经验去排查错误

3.4SpringBoot配置项目

springboot 配置mybatis

3.4.1 SpringBoot 项目结构

图中的5个文件和文件夹 可以先删掉 没用

static中的文件 不会在用户请求的时候，被总控制器给拦截。 总控制器MVC会将用户的请求给进行拦截，然后进行处理，但是放在static中的文件在用户请求的时候，不会被拦截，可以直接获取

springboot 不在使用jsp作为默认的动态模板，而是使用Thymeleaf 作为新的模板，优于jsp

application.properties 是开发者可以自定义进行修改的配置文件 利用数据库密码等 在这里修改配置

.properties 是使用键值对的数据配置

yml是使用层级目录进行配置

3.4.2 ⾃定义配置(整合MyBatis)

SpringBoot帮助我们完成通⽤性配置，但是像数据库连接地址、账号、密码等还是需要⼿动完成配置

修改mysql驱动的版本（选择性）

根据自己本地的项目数据库 情况 自己配置对应的数据库版本

在SpringBoot主配置⽂件 application.properties ⽂件中配置数据源及扫描包路径

在SpringBoot启动类通过 @MapperScan 注解指定DAO接⼝的包名

3.5启动SpringBoot项目

直接启动springboot 的启动类就可以实现项目启动【SpringBoot应⽤⾃带Servlet容器—Tomcat,因此⽆需进⾏额外的服务器配置，运⾏启动类即可启动⼀个 SpringBoot应⽤】

3.5.1 测试项目——insertUser

对数据表Users 增加一条数据 来测试整个SpringBoot项目

流程 先从数据库表开始，创建实体类，然后实现Dao层【创建UserMapper.xml文件】

遇到的bug

在编写SpringBoot测试的时候，配置项目中 路径写错了，在mapper后边多写了一个s 导致找不到mapper.xml 文件

**异步请求 **

==使用ajax 需要返回json对象的时候 需要增加@ResponseBody注解来实现数据 返回，不然会导致数据库中插入了数据，但是这里在前端进行页面跳转的时候没有页面返回，会报错404 错误==

==使用ajax请求 请求对象的时候 要加上@ResponseBody注解，如果使用url请求跳转 但是后端没有相应页面 也就是没有返回ModelAndView或者一个String数据 能够进行页面跳转 否则就要加上@ResponseBody注解==

如果整合第三方的工具 则需要稍加的配置数据

3.6 了解SpringBoot

3.6.1 SpringBoot 的Start

springboot 在一些组件配置上 提供了默认配置 但是在一些组件没有提供默认配置，需要自己手动提供——使用全局配置进行书写

像redis 是提供默认配置的，配置的是本地localhost，但是mybatis没有配置数据源 和 url，需要自己手动给填写数据源

server:
port: 9999
servlet:
context-path: /demo1

// 这里进行更改端口号 和项目名称 改成demo1 使用demo1进行访问

访问链接: http://localhost:9999/demo1/user/……

修改mybati驱动版本

小于8.0 版本 驱动是 com.mysql.jdbc.Driver

8.0以上的时候 需要加一个cj com.mysql.cj.jdbc.Driver
